# 信用评估系统 - Agent指南

## 项目概述
- **路径**: `~/credit-risk-system`
- **技术栈**: Python 3.13 + venv
- **核心功能**: 企业信用风险评估，整合天眼查数据、大众点评门店数据、内部交易数据
- **输出**: Excel信用评估报告

## 关键命令

```bash
# 激活环境（如果venv存在）
source .venv/bin/activate 2>/dev/null || true

# 生成详细评估报告
python3 scripts/generate_detailed_report.py

# 抓取天眼查真实数据
python3 scripts/fetch_tianyancha_real.py

# 抓取大众点评数据
python3 scripts/dianping_search.py "关键词"
```

## 系统架构

### 6维度评分卡（权重差异≤10%）
| 维度 | 权重 | 核心指标 |
|------|------|----------|
| 基础信用 | 18% | 成立年限、实缴资本、天眼评分(40%)、经营异常 |
| 公司规模 | 16% | 参保人数(50%)、关联公司数量/健康度 |
| 财务健康 | 16% | sudden_death_risk、自身风险数、缺省财务指标(10分) |
| 库存周转 | 16% | 库存周转天数(60%)、近3月销售环比 |
| 履约行为 | 18% | 回款准时率、逾期天数、被执行人次数、关联健康度 |
| 外部风险 | 16% | 诉讼数量、失信记录、股权冻结、大众点评门店数 |

### 规则引擎
- 头部连锁保底: 门店≥1000家 → 等级不低于A
- 微型企业否决: 门店<10家 → 等级最高C
- 按规模分层诉讼阈值

### 天眼评分调整
最终总分 = 原始分 × 0.3 + 天眼评分 × 0.7

## 重要约束
- **缺省分10分**: 无真实数据指标给10分（严格策略）
- **实缴资本**: 直接取自天眼查原始值，不通过"注册资本×比例"计算
- **门店规模系数**: <10家=0×, 50家=1×, 1000家=5×, 线性插值
- **库存周转**: 门店<5家时维度得0分
